## ğŸ¨ State Management Pattern

### Master Pages (e.g., Courier):
```typescript
const [companies, setCompanies] = useState<any[]>([]);  // Always array
const [loading, setLoading] = useState(true);
const [error, setError] = useState<string | null>(null);

// Fetch
const fetchCouriers = async () => {
  setLoading(true);
  try {
    const res = await fetch('/api/masters/courier');
    
    if (!res.ok) {
      setCompanies([]);  // Empty array on error
      return;
    }
    
    const data = await res.json();
    
    if (Array.isArray(data)) {
      setCompanies(data);  // Set data if valid array
    } else {
      setCompanies([]);    // Empty array if invalid
    }
  } catch (err) {
    setCompanies([]);      // Empty array on exception
  } finally {
    setLoading(false);
  }
};

// Add new (optimistic update)
const handleSubmit = async () => {
  const res = await fetch('/api/masters/courier', { method: 'POST', ... });
  const newRecord = await res.json();
  
  setCompanies(prev => [newRecord, ...prev]);  // Immediate UI update
};
```

**Key Points:**
- State always initialized as empty array
- Errors set state to empty array (not error object)
- Optimistic updates for instant feedback
- No `.filter()` errors possible

## ğŸ” Security Considerations

1. **JWT Token:**
   - Stored in cookie (httpOnly=false for client access)
   - 8-hour expiration
   - Verified on every API request

2. **Session Validation:**
   - Every protected API route validates session
   - Returns 401 if invalid/missing
   - Logs authentication state

3. **Team Isolation:**
   - Enforced at database query level
   - No client-side filtering
   - Super Admin bypass requires exact role match

4. **No Client Trust:**
   - API ignores client-sent userId/role
   - Always uses server-side session
   - Team filter constructed server-side
